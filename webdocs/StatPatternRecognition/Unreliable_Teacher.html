Learning with an unreliable teacher¶

In a typical classification setting, we are given feature data stored in a data matrix $X$ and labels for each observation stored in a vector $y$ (the training data). Our goal is simply to learn a way to predict the labels from the data so that we may classify new observations with unknown labels.

However, as addressed in a paper by Gabor Lugosi, our given training labels may not come from a reliable teacher. That is, a fraction of them may be incorrect due to corruption, whether from human error or noisy measurement devices. Intuitively, one may think that the best solution is to always seek the lowest error rates. However, it turns out that this is not the case as we shall see.

# Helper functions to simulate data
def get_data_class(label, n, mu, cov, p_mislabel):
    X = np.random.multivariate_normal(mu, cov, size=(n))
    Y = np.ones(n)*label
    Z = np.abs(np.random.binomial(1, 1-p_mislabel, size=n)-1+label)
    
    return X,Y,Z

def get_data(labels, ns, mus, cov, p_mislabels):
    X0,Y0,Z0 = get_data_class(labels[0], ns[0], mus[0], cov, p_mislabels[0])
    X1,Y1,Z1 = get_data_class(labels[1], ns[1], mus[1], cov, p_mislabels[1])
    
    X = np.vstack((X0,X1))
    Y = np.hstack((Y0,Y1))
    Z = np.hstack((Z0,Z1))
    
    indices = np.arange(X.shape[0])
    np.random.shuffle(indices)
    
    return(X[indices],Y[indices],Z[indices])

for _ in range(n_runs):
    # Simualate testing data
    X_test, Y_test, _ = get_data(
        [0,1], [n_test, n_test], [[-1,0],[1,0]], cov, [0,0]
    )

    # Balanced training data
    X_train_balanced, Y_train_balanced, Z_train_balanced = get_data(
        labels, [ns[-1], ns[-1]], [mu0, mu1], cov, balanced_err
    )
    # Unbalanced training data
    X_train_unbalanced, Y_train_unbalanced, Z_train_unbalanced = get_data(
        labels, [ns[-1], ns[-1]], [mu0, mu1], cov, unbalanced_err
    )
    for n in ns:
        clf_balanced.fit(X_train_balanced[:n], Z_train_balanced[:n])
        err = np.sum(np.abs(clf_balanced.predict(X_test) - Y_test)) / len(Y_test)
        scores_balanced[n].append(err)

        clf_unbalanced.fit(X_train_unbalanced[:n], Z_train_unbalanced[:n])
        err = np.sum(np.abs(clf_unbalanced.predict(X_test) - Y_test)) / len(Y_test)
        scores_unbalanced[n].append(err)

The data¶

We simulate a binary classification problem where observation $X_i \sim \mathcal{N}([-1, 0],I_2)$ is from class 0 and observation $X_i \sim \mathcal{N}([1, 0],I_2)$ is from class 1. In the balanced case, $40$% of both class labels are incorrect (i.e. label 0 becomes label 1 and vice versa) and in the unbalanced case, $20$% of class 0 and $30$% of class 1 are incorrect.

Misclassification rates¶

We train a logistic regression classifier which fits a regression model to estimate the log odds ratio of the two classes. Despite the unbalanced case having lower labels of mislabeling for both classes as compared to the balanced case, the classifier fit to the balanced mislabeled data generalizes better to new data.

Learned decision boundary¶

The optimal (Bayes) decision boundary simply classified points based on the sign of their first coordinate but is generally unknown. Above is plotted the learned decision boundaries from the classifiers in both cases. In the balanced case the optimal decision boundary is not biased towards either class amd so remains the same as the Bayes boundary, despite the mislabels. In the unbalanced case, we see that the decision boundary shifts away from the Bayes boundary and thus the classifier mislabels new observations in that region.
